[ALTO] Regex de LP muito abrangentes causam falsos positivos
Localização: StateMachine.js:59-63

Componente: Global Intent Detection

Problema: Os regex para detectar LPs (LP_AUTISMO, LP_DISLEXIA, etc.) são extremamente abrangentes:

LP_AUTISMO: /avalia[çc][aã]o\s+(de\s+)?autismo|...|\bautismo\b/i,
LP_DISLEXIA: /dislexia|troca\s+(b\/d|letras)|.../i,
A palavra \bautismo\b captura qualquer menção a autismo, mesmo em contextos como:

"meu filho NÃO tem autismo"
"descartaram autismo"
"quero saber se é autismo ou apenas atraso"
Isso faz o bot responder sobre avaliação de autismo quando o usuário pode estar descartando a possibilidade.

Impacto: Respostas descontextualizadas, experiência do usuário ruim, potencial perda de leads que se sentem "empurrados" para um diagnóstico que não querem.

Correção sugerida:

// Adicionar contexto negativo e positivo mais específico
LP_AUTISMO: /(?:vi\s+(?:no\s+site|sobre)|queria\s+(?:saber|entender)|tenho\s+dúvidas?\s+sobre|suspeito\s+que|acho\s+que\s+(?:meu|minha)|percebido\s+(?:sinais|comportamentos?)).*(?:autismo|tea|olha\s+nos\s+olhos|não\s+responde\s+ao\s+nome)/i,

// Ou criar validação adicional
export function detectGlobalIntent(text) {
    const lowerText = text.toLowerCase();
    
    for (const [intentName, regex] of Object.entries(GLOBAL_INTENTS)) {
        if (regex.test(text)) {
            // Validação de contexto negativo
            if (intentName.startsWith('LP_')) {
                const negativePatterns = [
                    /não\s+(?:tem|é|tenho)|descartaram|descartado|já\s+foi\s+(?:descartado|excluído)|não\s+(?:suspeito|acho)|só\s+queria\s+confirmar\s+que\s+não/i
                ];
                if (negativePatterns.some(p => p.test(lowerText))) {
                    return null; // Não detectar como LP se há contexto negativo
                }
            }
            return intentName;
        }
    }
    return null;
}
