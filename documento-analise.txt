Análise: Criação de Conteúdo com IA — Marketing Dashboard
Arquitetura Atual
O sistema possui dois pipelines paralelos de geração para Instagram que coexistem:

Sistema 1 (original): POST /instagram/generate → Bull queue → post.worker.js → instagramPostService.js
Sistema 2 (presets): POST /instagram/generate-with-layout → instagramController.gerarPostComLayout → JSON de presets
Para GMB e Facebook existe apenas o Sistema 1.

Problemas Encontrados
Bug crítico — promptBase indefinido
Em instagramPostService.js:222, a primeira tentativa com fal.ai usa promptBase (variável não existe; a correta seria prompt). Isso faz a Tentativa 1 sempre crashar silenciosamente e cair no Freepik, que tem um prompt hardcoded (lines 256-263) — sempre a mesma cena genérica de terapeuta, ignorando a especialidade e o tema customizado.

Seletor de Layout do Instagram está comentado
O código do seletor de layouts (MarketingDashboard.tsx:857-914) está todo em {/* */}. O Sistema 2 foi construído mas está inacessível pela UI.

Labels de modo enganosas
Os cards "Modo de geração" em linha 917-952 mostram 📸 Insta no modo Completo e 🔵 Google no modo Legenda — mesmo quando a aba ativa é Facebook. Confunde o usuário.

GMB tem agendamento, Instagram e Facebook não
O scheduling (📅 Agendar) aparece apenas para activeTab === 'gmb'. Posts de Instagram e Facebook só podem ser gerados imediatamente, sem agendamento.

max_tokens: 400 para legendas
Em instagramPostService.js:161, a legenda (pedida com "4-5 parágrafos curtos") tem limite de 400 tokens (~300 palavras). Legendas mais ricas e com maior estrutura ficam truncadas.

Opção "Gemini Nano" no select de providers
Aparece no dropdown da UI (linha 843) mas não existe implementado no chain de fallback de instagramPostService.js. Se selecionado, o sistema cai direto no Pollinations.

Melhorias de Criatividade e Qualidade
1. Tom de voz selecionável

Hoje todos os posts usam o mesmo tom emocional de dor/urgência. Adicionar opção: Educativo / Inspiracional / Urgência / Bastidores da clínica. O prompt do GPT já suporta isso, só precisa de um parâmetro a mais.

2. Variações automáticas (A/B)

Ao invés de gerar 1 post, gerar 2-3 variações da headline e legenda e deixar o usuário escolher a melhor antes de salvar.

3. Preview do prompt de imagem

Mostrar ao usuário o prompt que será enviado para a IA de imagem antes de gerar, com possibilidade de editar. Aumenta muito a qualidade das imagens.

4. Geração em lote semanal

Um botão "Planejar semana" que gera automaticamente posts para todas as especialidades com distribuição de funil ao longo dos 7 dias — já tem a lógica em createPostsForAllEspecialidades() no backend, falta expor na UI.

5. Score de engajamento estimado

Após gerar o texto, passar por um segundo prompt que avalia o post em 3 dimensões: Clareza, Urgência emocional, Call-to-action — e exibe um score simples (ex: 7/10) com sugestão de melhoria.

6. Carrossel para Instagram

O sistema só gera post de imagem única. Carrosséis têm 3x mais saves. Adicionando um modo carrossel que gera 3-5 slides com headline + texto por slide.

7. Hashtags por tendência de sazonalidade

O sistema fixa #anapolis #fonoaudiologia como hashtag. Nenhuma variação por época (volta às aulas, mês da criança, semana da audição). Um mapa de datas sazonais ligado ao tema customizado melhoraria o alcance orgânico.

8. Reativar layout selector e ativar via toggle

O Sistema 2 com os 5 presets JSON está pronto mas comentado. Bastaria um toggle na UI entre "Layout Automático" e "Preset Profissional".

    } = options;

    const prompt = temaCustom
      ? buildCustomPrompt(temaCustom, especialidadeId)
      : PROMPTS_ESPECIALIDADE[especialidadeId] || PROMPTS_ESPECIALIDADE.fonoaudiologia;

    logger.info(`[VEO SERVICE] 🎬 Iniciando geração — ${especialidadeId} — ${durationSeconds}s ${aspectRatio}`);
    logger.info(`[VEO SERVICE] Prompt (primeiros 120 chars): ${prompt.substring(0, 120)}...`);

    let operation;
    try {
      operation = await this.ai.models.generateVideos({
        model: 'veo-3.0-generate-preview',
        prompt,
        config: {
          resolution: '1080p',
          aspectRatio,
          durationSeconds,
          generateAudio: true  // Gera som ambiente natural
        }
      });
    } catch (err) {
      // Fallback para modelo sem audio se necessário
      logger.warn(`[VEO SERVICE] Tentativa 1 falhou, tentando sem áudio: ${err.message}`);
      operation = await this.ai.models.generateVideos({
        model: 'veo-3.0-generate-preview',
        prompt,
        config: {
          resolution: '1080p',
          aspectRatio,
          durationSeconds,
          generateAudio: false
        }
      });
    }

    // Polling até completar (~3-5 min)
    const MAX_WAIT_MS = 8 * 60 * 1000; // 8 minutos máximo
    const POLL_INTERVAL_MS = 15_000;   // 15s entre checks
    const startTime = Date.now();

    while (!operation.done) {
      if (Date.now() - startTime > MAX_WAIT_MS) {
        throw new Error('[VEO SERVICE] Timeout: vídeo não gerado em 8 minutos');
      }
      logger.info(`[VEO SERVICE] ⏳ Aguardando geração... (~${Math.round((Date.now() - startTime) / 1000)}s)`);
      await new Promise(resolve => setTimeout(resolve, POLL_INTERVAL_MS));
      await operation.poll();
    }

    if (operation.error) {
      throw new Error(`[VEO SERVICE] Erro da API Veo: ${operation.error.message}`);
    }

    const generatedVideo = operation.response?.generated_videos?.[0];
    if (!generatedVideo?.video) {
      throw new Error('[VEO SERVICE] Nenhum vídeo retornado pela API');
    }

    logger.info(`[VEO SERVICE] ✅ Vídeo gerado! Baixando...`);

    // Download do vídeo
    const videoBuffer = await this.ai.files.download({ file: generatedVideo.video });

    // Salvar temporário
    const tempPath = path.join('/tmp', `veo_${Date.now()}.mp4`);
    if (Buffer.isBuffer(videoBuffer)) {
      fs.writeFileSync(tempPath, videoBuffer);
    } else if (videoBuffer instanceof ArrayBuffer) {
      fs.writeFileSync(tempPath, Buffer.from(videoBuffer));
    } else {
      // Se for Uint8Array ou similar
      fs.writeFileSync(tempPath, Buffer.from(videoBuffer));
    }

    // Upload para Cloudinary
    logger.info(`[VEO SERVICE] ☁️ Enviando para Cloudinary...`);
    const cloudinaryResult = await cloudinary.v2.uploader.upload(tempPath, {
      resource_type: 'video',
      folder: 'fono-inova/ai-videos/veo',
      public_id: `veo_${especialidadeId}_${Date.now()}`,
      overwrite: false
    });

    // Limpar arquivo temporário
    try { fs.unlinkSync(tempPath); } catch { /* ignora */ }

    logger.info(`[VEO SERVICE] 🎉 Sucesso! URL: ${cloudinaryResult.secure_url.substring(0, 60)}...`);

    return {
      url: cloudinaryResult.secure_url,
      duration: cloudinaryResult.duration || durationSeconds,
      bytes: cloudinaryResult.bytes,
      provider: 'veo-3.1'
    };
  }
}
